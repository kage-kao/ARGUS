# ARGUS — Telegram бот для записи стримов (с поддержкой BuzzCast)

ARGUS принимает ссылку на стрим, пишет его до самого конца через `yt-dlp`,
режет на куски ≤199 МБ, **сжимает каждый через ezgif.com/video-compressor**
(вся тяжёлая работа сжатия на стороне ezgif), склеивает обратно
(`ffmpeg -c copy`, без потерь) и заливает результат в выбранный
файлообменник.

## 🆕 Новые функции

### 📺 BuzzCast интеграция
- Запись стримов BuzzCast по ID пользователя
- Автоматическая проверка статуса пользователя (в эфире/офлайн)
- Получение HLS URL стрима напрямую из BuzzCast API

### 📡 Система мониторинга
- Автоматическое отслеживание пользователей BuzzCast
- Проверка каждые 5 минут
- Автоматический старт записи при выходе в эфир
- Управление списком мониторинга через бота

### 🔐 Безопасность токенов
- Бот **всегда** запрашивает Telegram bot token при установке
- Нет сохранения токенов между установками
- Повышенная безопасность при переустановке

## Установка (Debian/Ubuntu/etc, под root)

```bash
curl -L https://raw.githubusercontent.com/kage-kao/ARGUS/main/setup.sh | bash
```

Скрипт:
1. Ставит `python3`, `ffmpeg`, `curl`, `ca-certificates`.
2. **Всегда запрашивает ввести Telegram bot token** прямо в терминале
   (получить можно у [@BotFather](https://t.me/BotFather)).
3. Скачивает исходники из репо, ставит pip-зависимости в venv.
4. Создаёт systemd-юнит `argus-bot.service` и запускает бота.

После установки — открой бота в Telegram и пришли `/start`.

### Альтернатива: токен через env

```bash
BOT_TOKEN=123:abc... bash -c "$(curl -L https://raw.githubusercontent.com/kage-kao/ARGUS/main/setup.sh)"
```

## Использование

### Основные команды

| Команда | Что делает |
|---------|------------|
| `/start` | приветствие с быстрыми кнопками |
| `/status` | список твоих активных задач с ID и прогрессом |
| `/cancel <job_id>` | отменить задачу: SIGTERM yt-dlp/ffmpeg + cancel asyncio-task |
| `/buzzcast <userId>` | записать стрим BuzzCast пользователя |
| `/monitoring` | показать список мониторинга |
| `/monitoring <userId>` | добавить пользователя BuzzCast в мониторинг |

### 📺 Запись обычных стримов

1. `/start` — приветствие.
2. Пришли боту ссылку на стрим (`m3u8` / `flv` / любой URL для `yt-dlp`).
3. Выбери файлообменник: **Ranoz** (4.99 ГБ), **Tempshare** (2 ГБ) или **оба сразу**.
4. Выбери **качество сжатия**: 🎬 High (720p · 1500 kbps), 📺 Medium
   (480p · 800 kbps), 📱 Low (360p · 400 kbps), 🪶 Tiny (240p · 200 kbps).
5. Если выбран Tempshare — выбери срок хранения: **1 / 3 / 7 дней**.
6. Бот пишет стрим **до конца** (это могут быть часы). По завершении пришлёт
   ссылки. Файлы больше лимита автоматически режутся и ты получаешь несколько
   ссылок.

### 📺 Запись стримов BuzzCast

1. Получи ID пользователя BuzzCast (например, `12345678`)
2. Отправь боту `/buzzcast 12345678`
3. Бот проверит, в эфире ли пользователь:
   - ✅ **В эфире** → предложит выбрать параметры записи (как обычный стрим)
   - 💤 **Офлайн** → предложит добавить в мониторинг
4. Выбери хостер, качество и срок хранения
5. Запись начнётся автоматически

### 📡 Мониторинг пользователей BuzzCast

**Добавить в мониторинг:**
1. `/monitoring 12345678` - добавить пользователя BuzzCast
2. Выбери параметры записи (хостер, качество, срок)
3. Мониторинг активирован!

**Управление мониторингом:**
1. `/monitoring` - показать список всех отслеживаемых пользователей
2. Нажми кнопку "🗑 Удалить" чтобы убрать из мониторинга

**Как работает:**
- Бот проверяет каждые **5 минут** статус всех пользователей в списке
- При обнаружении начала стрима автоматически начинает запись
- Уведомляет тебя в чат о начале записи
- После завершения стрима пришлёт ссылки на скачивание
- Продолжит мониторить пользователя для следующих стримов

### 🚀 Быстрые кнопки

После `/start` появляются быстрые кнопки:
- **📊 Мои задачи** - показать активные записи
- **📡 Мониторинг** - список отслеживаемых пользователей
- **📺 BuzzCast запись** - инструкция по записи BuzzCast

## Файлообменники

| Сервис | Лимит файла | Хранение | Особенность |
|--------|------------|----------|-------------|
| **Ranoz** (`ranoz.gg`) | 4.99 ГБ | бессрочно | Видео-расширения блокируются → грузим как `.dat`. После скачивания переименуй в `.mp4`. |
| **Tempshare** (`tempshare.su`) | 2 ГБ | 7 дней | Прямые ссылки, всё прозрачно. |

Лимиты обходятся автоматическим демукс-сплитом (`ffmpeg -c copy`) — без
реэнкода и без потерь качества.

## Параметры (env / `.env`)

| Переменная | Дефолт | Что значит |
|------------|--------|------------|
| `EZGIF_FORMAT` | `mp4` | формат выхода ezgif |
| `DEFAULT_QUALITY` | `low` | пресет по умолчанию (`high`/`medium`/`low`/`tiny`) |
| `EZGIF_PARALLEL` | `2` | параллельных компрессий ezgif |
| `UPLOAD_PARALLEL` | `2` | параллельных загрузок на хостер |

Качество выбирается каждый раз через инлайн-кнопки. Срок хранения Tempshare
(1/3/7 дней) — тоже.

Изменить — отредактируй `/opt/argus-bot/.env`, потом
`systemctl restart argus-bot`.


```
├── __main__.py         # python -m argus_bot
├── bot.py              # aiogram handlers + UI + monitoring loop
├── buzzcast_client.py  # BuzzCast API client (NEW)
├── pipeline.py         # main flow: record → split → compress → concat → upload

├── compressor.py       # клиент ezgif.com/video-compressor
├── uploader.py         # Ranoz + Tempshare
├── storage.py          # SQLite-хранилище задач + мониторинга (UPDATED)
└── config.py           # env-конфиг + пресеты качества
```

### Новые компоненты

#### BuzzCast Client (`buzzcast_client.py`)
- Анонимный tourist-доступ к BuzzCast API
- Поиск пользователей по ID
- Получение HLS URL активных стримов
- Автоматическое получение tourist токенов

#### Система мониторинга
- Таблица `monitoring` в SQLite
- Фоновая задача проверки каждые 5 минут
- Автоматический запуск записи при обнаружении стрима
- Персистентное хранение настроек для каждого отслеживаемого пользователя

## База данных

### Таблица `jobs` (записи)
- Хранит все задачи записи
- Статусы: pending/recording/processing/uploading/done/failed/cancelled/interrupted
- Ссылки на скачивание после завершения

### Таблица `monitoring` (мониторинг) 🆕
- Список отслеживаемых пользователей BuzzCast
- Настройки записи для каждого пользователя
- Timestamp последней проверки
- ID активной задачи записи


## Технические детали BuzzCast интеграции

### API эндпоинты
- `POST /api/user/pwa/event` - получение tourist токена
- `POST /api/token/social/search/findUserByNickNameOrId` - поиск пользователя
- `POST /api/token/live/info/getChannelListByRecommend` - список live каналов

### Процесс записи BuzzCast
1. Получение tourist токена (анонимный доступ)
2. Поиск пользователя по ID → проверка `liveState`
3. Если в эфире: получение списка live каналов
4. Фильтрация по `userId` для нахождения нужного стрима
5. Извлечение HLS URL (приоритет: hlsUrl > playUrl > videoUrl)
6. Запуск стандартного пайплайна ARGUS для записи

### Мониторинг
- Интервал проверки: **5 минут** (300 секунд)
- Автоматический перезапуск при обнаружении стрима
- Уведомления в Telegram при начале записи
- Продолжение мониторинга после завершения записи
