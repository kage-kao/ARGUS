## ARGUS
ARGUS - сервис, который сам записывает и сохраняет для вас прямые трансляции (стримы).

![alt text](https://i.imgur.com/mxvw9Ue.png)

# Как это работает:

1. Вы просто даёте ему ссылку на нужный стрим.
(Пример ссылки: https://pull.buzzcast.com/live/23331_3dna00j.flv?txSecret=c04ad576bdcfa9bbf6ab799240ea0434&txTime=691DB5CF&ud=8&id=b5845eb407e52468ab53878560bbde33)

2. ARGUS сам, в фоновом режиме, записывает видео от начала и до самого конца. Вам не нужно держать компьютер включенным или следить за процессом.

3. Когда стрим закончится, ARGUS:
   - нарежет запись на **валидные mp4-сегменты до 199 МБ** каждый (через `ffmpeg -c copy` — это демукс без реэнкода, практически не нагружает ваш сервер);
   - **сожмёт каждый сегмент максимально через ezgif** (`https://ezgif.com/video-compressor`) — сторонний энкодер берёт всю тяжёлую работу сжатия на себя, ресурсы вашего сервера не тратятся;
   - склеит сжатые сегменты обратно в один файл (`ffmpeg -c copy`);
   - загрузит готовую запись сразу на **два файлообменника** и добавит на главную страницу обе ссылки для скачивания.

# Файлообменники

| Сервис | Лимит файла | Хранение | Особенность |
|--------|------------|----------|-------------|
| **Ranoz** (`ranoz.gg`) | 4,99 ГБ | — | Блокирует видео по расширению, поэтому запись льётся как `.dat`. После скачивания переименуйте файл в `.mp4`. Файлы больше 4,99 ГБ автоматически режутся на части. |
| **Tempshare** (`tempshare.su`) | 2 ГБ | 7 дней | Прямые ссылки `url` и `raw_url`. Файлы больше 2 ГБ автоматически режутся на части. |

Если итоговая запись превышает лимит сервиса — ARGUS сам нарежет её на части (через `ffmpeg -c copy`) и загрузит каждую часть отдельно, выдав все ссылки.

# Параметры сжатия ezgif (в `record_and_upload.sh`)

```bash
EZGIF_RES="640x360"     # целевое разрешение
EZGIF_BITRATE="300"     # kbps
EZGIF_FORMAT="mp4"
EZGIF_MAX=175000000     # ~175 МБ — порог нарезки сегментов (с запасом под лимит ezgif 200 МБ)
```

Для более агрессивного сжатия уменьшите `EZGIF_RES` (например `426x240`) и `EZGIF_BITRATE` (например `200`). Все сегменты должны сжиматься с одинаковыми параметрами — иначе склейка `ffmpeg -c copy` может рассинхронизировать видео, и ARGUS автоматически откатится на re-encode.

# Установка:

```bash
curl -L https://raw.githubusercontent.com/kage-kao/ARGUS/main/setup.sh | bash
```

После установки сервис доступен на `http://<IP-сервера>:5000`.

# Логи

```bash
journalctl -u stream-recorder -f
```

В логах видно каждый этап: запись → нарезка сегментов → сжатие каждого через ezgif → склейка → загрузка на Ranoz и Tempshare с готовыми ссылками.
